# MusicPI v0.1.0-beta.1

The first public beta of **MusicPI** — one Raspberry Pi + Native Instruments
Maschine MK3 that boots into a Tracktion-based DAW (**MusicPI**) or **MixxxDJ**,
chosen at boot. This is an early beta: expect rough edges, and please report
what you find.

## What's in this image

- A single flashable Raspberry Pi OS Lite (arm64) image with **both modes**
  installed and selected from the MK3 screens at boot.
- **MusicPI** mode — 16-pad sampler, pattern sequencing, song arrangement,
  groups, mixer, sample browsing/editing, FourOsc synth + VST3/LV2 hosting, and
  `.mpi` project save/load.
- **MixxxDJ** mode — Mixxx with full MK3 control (decks, EQ, pads for
  hotcues/loops, and the hardware screens mirroring the UI).
- First-boot auto-provisioning: the image expands to fill your card and creates
  two data partitions (`MIXXX_LIBRARY`, `MPI_SAMPLES`).

## Install

1. Download `mpi-station-YYYYMMDD.img.xz` below and verify it:
   ```bash
   sha256sum -c mpi-station-YYYYMMDD.img.xz.sha256
   ```
2. Flash with [Raspberry Pi Imager](https://www.raspberrypi.com/software/),
   balenaEtcher, or `dd`.
3. Boot with the MK3 connected; wait for the selector, then pick a mode.

Full walkthrough: [Getting started](../getting-started.md).

**Default login:** `mpi` / `maschinepi` — change the password before putting the
device on an untrusted network.

## Known issues (beta)

This is an early beta; the following are known and being worked on:

- **MusicPI:** undo does not yet cover mixer gain/pan or step-velocity edits;
  first launch can be slow with very large sample libraries.
- **MixxxDJ:** if the desktop session restarts, the MK3 screens may freeze —
  reboot to recover. Mouse mode is CPU-heavy and is best avoided during a live
  set. The on-screen settings overlay is early; prefer the pads/knobs for live
  control.
- General: this is a first beta — save often, and back up anything important
  before re-flashing.

See the [beta-blocker triage](../beta-blocker-triage.md) for the full list and
status.
