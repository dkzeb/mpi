# MusicPI v0.9.0-beta.2

The first public beta of **MusicPI** — one Raspberry Pi + Native Instruments
Maschine MK3 that boots into a Tracktion-based DAW (**MusicPI**) or **MixxxDJ**,
chosen at boot. This is a late-stage beta on the path to v1: expect some rough
edges, and please report what you find.

## What's in this image

- A single flashable Raspberry Pi OS Lite (arm64) image with **both modes**
  installed and switched by an on-screen selector at boot.
- **MusicPI** mode — 16-pad sampler, pattern sequencing, song arrangement,
  groups, mixer, sample browsing/editing, FourOsc synth + VST3/LV2 hosting, and
  `.mpi` project save/load.
- **MixxxDJ** mode — Mixxx with full MK3 control (decks, EQ, pads for
  hotcues/loops, and the hardware screens mirroring the UI).
- First-boot auto-provisioning: the image expands to fill your card and creates
  two data partitions (`MIXXX_LIBRARY`, `MPI_SAMPLES`).
- Beta 2 fixes the partition-growth ordering found during real-card validation,
  so the Mixxx library and MusicPI samples each receive half of the available
  space after the base OS.

## Install

1. Download `mpi-station-*.img.xz` and its `.sha256` from the assets below, then
   verify:
   ```bash
   sha256sum -c mpi-station-*.img.xz.sha256
   ```
2. Flash it to a 16 GB+ SD card with
   [Raspberry Pi Imager](https://www.raspberrypi.com/software/) (*Use custom
   image*), balenaEtcher, or `dd`.
3. Boot the Pi with the MK3 connected. On first boot the image expands and sets
   up its data partitions (this takes a few minutes, once).
4. When the on-screen selector appears, pick **MusicPI** or **MixxxDJ**. Reboot
   to return to the selector and switch.

Full walkthrough: [Getting started](../getting-started.md).

**Default login:** `mpi` / `maschinepi` — change the password before putting the
device on an untrusted network.

## Known issues (beta)

The following beta issues are known and being worked on:

- **MusicPI:** undo does not yet cover mixer gain/pan or step-velocity edits;
  first launch can be slow with very large sample libraries.
- **MixxxDJ:** if the desktop session restarts, the MK3 screens may freeze —
  reboot to recover. Mouse mode is CPU-heavy and best avoided during a live set.
  The on-screen settings overlay is early; prefer the pads/knobs for live control.
- General: this is a first beta — save often, and back up anything important
  before re-flashing.

## Verifying the download

The image's SHA-256 is in the attached `.sha256` file; `sha256sum -c` (above)
checks it. Raspberry Pi Imager also verifies the write automatically.
